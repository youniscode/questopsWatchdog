## Phrases to avoid

- "Guaranteed uptime" — dishonest, no tool guarantees uptime
- "Enterprise-grade" — misleading for a PowerShell script collection
- "AI-powered" — not applicable, the tool runs deterministic checks
- "Set and forget" — scheduled scans still need occasional review
- "Monitor everything" — Windows only, limited to configured checks
- "Real-time" — scans are periodic (configurable interval), not streaming

---

## Honest positioning notes

- **This is a simple tool that does a few things well.** Do not oversell it as a full monitoring platform.
- **Windows-only is a constraint, not a feature.** Acknowledge it clearly so Linux users are not misled.
- **Free is the primary value.** The paid audit is an optional add-on service, not a paywall.
- **No fake social proof.** Do not fabricate testimonials, download counts, or usage statistics.
- **Pricing transparency.** State clearly that prices are suggested and adjustable. No hidden fees.
- **No false urgency.** Avoid "limited time offer", "only X spots left", or similar pressure tactics.
- **Acknowledge limitations.** The tool is not a service, does not auto-fix issues, and requires manual config per server.

---

## Technical SEO notes

- Page should be static HTML (no JavaScript framework required for rendering).
- Include structured data (JSON-LD) for SoftwareApplication and Offer.
- Canonical URL should point to the live site or GitHub repo.
- Open Graph and Twitter Card meta tags for social sharing.
- Robots.txt should not block any content sections.
- Sitemap should include all major sections (hero, features, pricing, FAQ).

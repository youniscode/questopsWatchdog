# QuestOps Watchdog v0.5.0 — Local Dashboard Release

**Status:** Official release notes

---

## Summary

This release adds a local HTML dashboard to QuestOps Watchdog — a self-contained, zero-dependency static HTML page that provides a visual overview of your latest health scan results and history. The dashboard runs entirely locally, uses inline CSS only (no JavaScript), and works offline.

The dashboard is the result of cumulative milestones v0.5.0 through v0.5.9. This release packages all dashboard work into a single official release.

## What is new

- **Local static dashboard exporter** — `export_questops_dashboard.ps1` reads your JSON scan report and optional history, then generates a standalone HTML file
- **CSS-only filter tabs** — All Servers / Passing / Failing status filters, no JavaScript required
- **Polished summary cards** — total servers, passed, failed, failed servers, history loaded
- **Server status table** — per-check PASS/FAIL badges for folder, disk, process, network, logs, and resources
- **Failed server detail section** — per-category explanations of what failed
- **History trend metrics:**
  - Pass/fail counts and pass/fail rates
  - Current and longest pass/fail streaks
  - Timeline pills for the most recent 20 scans
  - Repeated failure insights — top 5 most-failing servers
- **Safe error pages** — missing or malformed reports produce a self-contained error page instead of a crash (exit 2/3)
- **Warning boxes** — partial or missing optional fields produce a yellow warning box; the dashboard still renders
- **Responsive and print-friendly** — works on smaller screens, prints with hidden controls and page-break avoidance
- **Release readiness and validation docs** — internal release preparation, decision planning, and final validation documentation

## Security / privacy

- Dashboard runs entirely locally — no data is sent anywhere
- No telemetry, no phone-home, no analytics
- No JavaScript — all interactivity is CSS-only
- No external dependencies (no CDN, no fonts, no images) — works offline
- All string values HTML-encoded before rendering
- Discord webhook URL patterns redacted as `[REDACTED]`
- Generated dashboard files are gitignored and excluded from the client package
- Internal planning and release docs are excluded from the client package
- No webhook URLs, no local configs, no generated reports, no internal docs in the client package

## Compatibility

- **Windows + PowerShell 5.1 only**
- No config schema changes required — existing `servers.example.json` and `servers.local.json` configs from v0.4.9 remain fully compatible
- Works with JSON reports generated by `questops_scan.ps1` (history requires `questops_run.ps1`)

## Known limitations

- Not a live dashboard — static HTML snapshot, regenerate after each scan
- No charts or graphs — trend summary uses text, tables, and CSS pills
- No browser-side auto-refresh — all data is baked in at generation time
- Screenshots are not included in this release — a capture plan exists in `docs/dashboard/DASHBOARD_SCREENSHOT_PASS.md`

## Upgrade notes

1. Pull the latest code: `git pull origin main`
2. Run a scan: `powershell -NoProfile -ExecutionPolicy Bypass -File scripts\questops_run.ps1 -NoAlert`
3. Generate the dashboard: `powershell -NoProfile -ExecutionPolicy Bypass -File scripts\export_questops_dashboard.ps1`
4. Open `reports\questops-dashboard.html` in any browser

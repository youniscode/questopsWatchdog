# QuestOps Watchdog — Outreach Tracker Template

Copy this table markdown to a local or private file (not in the repository) if using real contact data.

## Columns

| Date | Platform | Community / context | Contact type | Message used | Response | Status | Follow-up date | Notes |
|------|----------|-------------------|-------------|-------------|----------|--------|---------------|-------|

## Status options

| Status | Meaning |
|--------|---------|
| Not contacted | Identified as potential contact but not yet reached |
| Contacted | First message sent |
| Replied | Received a response of any kind |
| Interested | Expressed interest in the tool or audit |
| Audit package sent | Client-ready package has been shared |
| Results received | Audit results bundle has been received |
| Audit delivered | Audit summary has been sent to the client |
| Closed won | Audit completed successfully, positive outcome |
| Closed lost | Declined or no longer relevant |
| Follow-up later | Not ready now but may be interested in the future |

## Rules

- **Do not store private sensitive data** in the tracker. No real names beyond what is needed for follow-up (handle/username is sufficient).
- **Do not paste private messages** into the repository. If you need to reference a conversation, use a sanitised summary.
- **Keep this as a local/private copy** if using real contacts. The template in the repository is for reference only.
- **Do not store webhook URLs, passwords, tokens, IPs, or any credentials** in the tracker.
- **Respect opt-out requests.** If someone asks not to be contacted, mark as "Closed lost" and do not follow up again.

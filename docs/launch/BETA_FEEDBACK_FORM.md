# QuestOps Watchdog — Beta Feedback Form

Copy and paste these questions after delivering a beta audit. The goal is to learn what works and what needs improvement.

---

## Feedback questions

### 1. Tool experience

Was the tool easy to run? (Yes / Mostly / No — please explain)

### 2. Report clarity

Did the report (JSON or HTML) make sense to you? (Yes / Mostly / No — what was confusing?)

### 3. Confusion points

What confused you the most? (Config format, command parameters, report output, anything else)

### 4. Issue identification

Did the scan help identify an issue you were not already aware of? (Yes / No — if yes, what was it?)

### 5. Likelihood to use again

Would you use QuestOps Watchdog again? (Definitely / Probably / Probably not / Definitely not)

### 6. Willingness to pay

Would you pay for this service (paid audit)? (Yes / Maybe / No)

### 7. Perceived fair price

What price would feel fair for a manual audit of your server setup? (Open answer)

### 8. Anonymous quote

Can this feedback be quoted anonymously? (Yes / No)

### 9. Attributed quote

Can this feedback be quoted with attribution (your name/handle)? (Yes / No — if yes, how would you like to be credited?)

---

## After collecting feedback

- [ ] Add anonymised insights to the project notes (without identifying the respondent)
- [ ] Note any feature requests or improvement ideas
- [ ] If a bug was reported, log it and fix it
- [ ] If feedback is positive and quotable, save it for future use (with permission)
- [ ] Thank the respondent for their time

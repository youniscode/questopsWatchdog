## Fields explained

| Field | Purpose |
|-------|---------|
| `generated_at_utc` | When the manifest was generated |
| `package_type` | Always `"audit-results"` |
| `product` | Product name for identification |
| `version` | Version from `VERSION` file |
| `files_included` | List of files in the bundle (forward-slash paths) |
| `warnings` | Non-fatal issues during packaging |
| `included_config` | Whether the client config was included |
| `included_log` | Whether the run log was included |

## Notes

- All filenames, paths, and data shown are **fictional**.
- The manifest is generated inside the zip and is not written to disk.
- The manifest is always the last entry added to the zip.
